## What this repo is

Astro + Starlight documentation site for the [Breadcrumbs Obsidian plugin](https://github.com/michaelpporter/breadcrumbs). Content is authored in Obsidian via Vault CMS and published by pushing to git.

## Commands

```bash
npm run dev       # dev server at localhost:4321
npm run build     # build to ./dist/
npm run preview   # preview built site
```

No test suite. TypeScript check: `npm run astro check`.

## Architecture

**Content location**: All docs live in `src/content/docs/`. Astro/Starlight serves files from there as routes — file path = URL path.

**Content format**: Markdown files with YAML frontmatter. Required frontmatter field: `title`. Files without `title` will fail the build. Obsidian-style wikilinks (`[[Note]]`) and callouts (`> [!INFO]`) are used in source but may need Starlight-compatible equivalents if they don't render correctly.

**Astro config** (`astro.config.mjs`): Sidebar is partially hardcoded, partially autogenerated via `autogenerate`. Adding a new top-level section requires updating the sidebar config here.

**Content config** (`src/content.config.ts`): Single `docs` collection using Starlight's `docsLoader()` and `docsSchema()`.

**Vault CMS** (`src/content/_GUIDE.md`, `src/content/_bases/`): Obsidian plugin that manages content. The `_bases/` folder contains Obsidian Bases views (`.base` files) for content management UI — not served by Astro.

**Images**: Referenced as Obsidian wikilinks (`![[Images/foo.png]]`) in source files. Stored in `src/content/docs/Images/`.

## Content structure

```
src/content/docs/
├── index.mdx              # Homepage (Starlight splash template)
├── Concepts.md            # Core concepts reference
├── Edge Fields.md
├── Field Groups.md
├── Note Attributes.md
├── Announcements/         # Release notes
├── Commands/              # Plugin commands
├── Explicit-Edge-Builders/
├── Implied-Edge-Builders/
├── Views/                 # Codeblocks, Matrix, Page, Tree, Trail views
├── Suggesters/
├── guides/                # How-to guides
└── Images/                # Static images
```

## Key constraints

- Starlight requires `title` in frontmatter for every `.md`/`.mdx` file in `docs/`
- `src/content/_bases/` and `src/content/.obsidian/` are Obsidian-specific — don't modify
- Publishing = `git push`; no separate deploy step
